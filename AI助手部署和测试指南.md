# AI助手智能记录功能 - 部署和测试指南

## 📦 部署步骤

### 1. 上传云函数（必须！）

**重要**：必须重新上传云函数才能应用最新的系统提示词和逻辑。

1. 打开微信开发者工具
2. 在左侧目录树中找到 `cloudfunctions/callSiliconFlowAI` 文件夹
3. **右键点击** `callSiliconFlowAI` 文件夹
4. 选择 **"上传并部署：云端安装依赖"**
5. 等待部署完成（通常需要1-2分钟）

### 2. 验证云函数版本

1. 打开浏览器，登录 [微信云开发控制台](https://console.cloud.tencent.com/tcb)
2. 进入你的云环境
3. 点击 **云函数 → callSiliconFlowAI**
4. 查看 **最后更新时间**，确认是最新的（应该是今天 2025-12-08）
5. 点击 **查看日志**，确保没有报错

### 3. 检查数据库集合

确保以下数据库集合已创建（如果没有，需要手动创建）：

- `bloodTests` - 血常规
- `liverFunctionTests` - 肝功能
- `kidneyFunctionTests` - 肾功能
- `bloodSugars` - 血糖
- `bloodOxygens` - 血氧
- `bloodPressures` - 血压
- `ebvRecords` - EB病毒
- `cmvRecords` - 巨细胞病毒
- `ldhRecords` - LDH

**创建方法**：
1. 云开发控制台 → 数据库
2. 点击 **"新建集合"**
3. 输入集合名称（如 `bloodTests`）
4. 点击确定

### 4. 建立数据库索引（推荐）

为每个集合创建索引，提升查询性能：

```javascript
// 在云开发控制台的数据库管理中，为每个集合创建索引
索引字段：openid + profileId + date
索引类型：升序
```

## 🔍 故障排查

### 问题1：AI不识别数据

**可能原因**：
- 云函数未更新到最新版本
- 系统提示词未生效

**解决方法**：
1. 重新上传云函数（步骤1）
2. 清除小程序缓存：开发者工具 → 清除缓存 → 重新编译
3. 查看云函数日志，确认调用成功

### 问题2：JSON代码块显示给用户

**可能原因**：
- `utils/markdown.js` 未更新

**解决方法**：
1. 确认 `utils/markdown.js` 第18-19行包含JSON移除代码：
   ```javascript
   // 🔥 移除JSON代码块
   html = html.replace(/```json\s*[\s\S]*?```/g, '');
   ```
2. 重新编译项目

### 问题3：数据未保存到数据库

**可能原因**：
- 数据库集合不存在
- 没有 openid 或 profileId
- 权限不足

**排查步骤**：

**步骤1：检查是否登录**
```javascript
// 在浏览器Console中执行
const app = getApp();
console.log('openid:', app.getOpenIdIfLoggedIn());
console.log('profileId:', app.getCurrentProfileId());
```

如果返回空，说明未登录或未选择档案：
1. 进入 **我的** 页面
2. 完成登录
3. 确保有默认档案

**步骤2：检查控制台日志**

在微信开发者工具的Console中查看：
```
✅ 健康数据已保存: { dataType: 'bloodTest', collection: 'bloodTests', values: {...} }
```

如果看到 `❌ 解析健康数据失败`，说明：
- AI未输出JSON格式
- JSON格式不正确
- 数据解析失败

**步骤3：手动查询数据库**

在云开发控制台 → 数据库 → bloodTests 集合中查询：
```javascript
{
  "source": "ai_assistant",
  "date": "2025-12-08"
}
```

如果有数据，说明保存成功。

**步骤4：检查云函数日志**

云开发控制台 → 云函数 → callSiliconFlowAI → 日志

查找关键信息：
- `云函数被调用` - 确认调用成功
- `AI回复成功` - 确认AI响应正常
- 错误信息 - 定位具体问题

### 问题4：Toast提示不显示

**可能原因**：
- 数据未成功保存
- parseAndSaveHealthData 函数未执行

**解决方法**：
1. 查看Console日志中是否有 `✅ 健康数据已保存`
2. 检查 `pages/ai-assistant/index.js` 第318行：
   ```javascript
   this.parseAndSaveHealthData(result.content, openid);
   ```
   确认该行存在且未被注释

### 问题5：数据格式错误

**典型错误**：
```
血常规记录中 wbc、plt、hgb、neut 字段为 null 或空
```

**原因**：
AI输出的JSON键名不正确（如使用大写WBC而不是小写wbc）

**解决方法**：
系统提示词已明确要求：
> values: 指标键值对（键名必须小写，如wbc、plt、hgb、neut）

如果仍有问题，检查云函数的系统提示词第94行。

## 📊 数据验证

### 查看保存的数据

1. 云开发控制台 → 数据库 → 选择集合（如 `bloodTests`）
2. 查询条件：
   ```json
   {
     "source": "ai_assistant",
     "date": "2025-12-08"
   }
   ```
3. 确认以下字段：
   - `openid` - 用户标识
   - `profileId` - 档案ID
   - `date` - 日期（YYYY-MM-DD）
   - `wbc`、`plt`、`hgb`、`neut` - 具体数值
   - `source: "ai_assistant"` - 来源标记
   - `createTime` - 创建时间

### 在健康档案中查看

1. 小程序中进入 **健康档案** 页面
2. 选择日期范围：今天
3. 选择数据类型：血常规
4. 查看是否有今天的记录
5. 点击记录查看详情

## 💡 使用技巧

### 技巧1：口语化输入

支持各种自然表达：
- ✅ "白细胞3.6"
- ✅ "我的白细胞是3.6"
- ✅ "今天血常规：白细胞3.6..."
- ✅ "白细胞3.6，血小板120"

### 技巧2：批量输入

一次性输入所有数据：
```
白细胞3.2，血小板80，血红40，中性粒80
```

### 技巧3：分步输入

先输入部分，AI会提示补充：
```
用户："白细胞3.6"
AI："还需要血小板、血红蛋白、中性粒细胞"
用户："血小板120，血红110，中性粒2.1"
AI："好的，已为您记录..."
```

### 技巧4：查询功能

除了记录，AI还可以回答健康问题：
```
用户："白细胞低怎么办？"
AI：[提供详细的健康建议]
```

## 🎯 性能优化

### 云函数超时设置

确认云函数配置文件 `config.json` 中的超时时间：
```json
{
  "timeout": 60,
  "envVariables": {},
  "runtime": "Nodejs16.13",
  "memorySize": 256
}
```

### API限流

硅基流动API有调用限制：
- 免费版：每分钟60次
- 超出限制会返回429错误

如遇频繁调用失败，建议：
1. 升级API套餐
2. 添加本地缓存
3. 限制用户调用频率
