# AI助手智能数据记录功能说明

## 📋 功能概述

AI助手"暖白记事本"现在支持智能识别用户的健康数据输入，并自动保存到对应的健康档案中。

## 🚀 部署步骤

### 1. 上传云函数
1. 在微信开发者工具中，右键点击 `cloudfunctions/callSiliconFlowAI` 文件夹
2. 选择"上传并部署（云端安装依赖）"
3. 等待部署完成

### 2. 验证部署
- 打开云开发控制台 → 云函数
- 确认 `callSiliconFlowAI` 函数版本已更新
- 查看日志确保没有错误

## ✨ 功能特性

### 支持的数据类型

| 数据类型 | 必选项 | 示例输入 |
|---------|--------|---------|
| 血常规 | WBC、PLT、HGB、NEUT（4项必选） | "白细胞3.6，血小板120，血红110，中性粒2.1" |
| 肝功能 | ALT、AST、TBIL、DBIL（4项必选） | "ALT 45，AST 38，总胆红素12，直接胆红素4" |
| 肾功能 | CR 或 BUN（至少1项） | "肌酐85，尿素氮6.5" |
| 血糖 | bloodSugar | "血糖5.2" |
| 血氧 | spo2 | "血氧98，心率72" |
| 血压 | 收缩压、舒张压（2项必选） | "血压130/80" |
| EB病毒 | ebvDna | "EBV-DNA 1200" |
| 巨细胞病毒 | hcmvDna | "CMV-DNA 800" |
| LDH | ldh | "LDH 180" |

### 智能识别能力

✅ **支持的表达方式**：
- 标准格式："白细胞3.6，血小板120"
- 口语化："我的白细胞是3.6"
- 简洁表达："白细胞3.6"
- 带日期："今天血常规：白细胞3.6..."

✅ **别名识别**：
- 白细胞 = WBC
- 血小板 = PLT
- 血红蛋白 = 血红 = HGB
- 中性粒细胞 = NEUT
- 肌酐 = CR
- 尿素氮 = BUN

### 必选项验证

当用户提供的数据**不完整**时，AI会友好提示：

**示例1：血常规数据不完整**
```
用户："白细胞3.6"
AI："您只提到了白细胞，血常规记录还需要：血小板、血红蛋白、中性粒细胞这3项数据。请问您的其他数据是多少呢？"
```

**示例2：肝功能数据不完整**
```
用户："ALT 45，AST 38"
AI："肝功能记录需要这4项数据：ALT、AST、总胆红素、直接胆红素。您可以再告诉我总胆红素和直接胆红素的数值吗？"
```

### 自动保存流程

当用户提供**完整数据**时：

1. **AI确认**："好的，已为您记录今天的血常规数据 ✓"
2. **输出JSON**（后端处理，用户不可见）：
   ```json
   {
     "dataType": "bloodTest",
     "date": "2024-01-15",
     "values": {
       "wbc": 3.6,
       "plt": 120,
       "hgb": 110,
       "neut": 2.1
     }
   }
   ```
3. **系统提示**："✅ 血常规数据已成功保存到您的健康档案"
4. **Toast提示**："血常规已自动保存"

## 🎯 AI行为规则

### 数据记录模式
- **仅在数据完整时**才输出JSON并保存
- **数据不完整时**提示用户补充，不保存
- 保存成功后给出简短确认
- 不过度解读数据，不主动给出医学建议

### 健康咨询模式
- 用温暖、专业的语言回答问题
- 提供实用的健康建议和护理指导
- 重要提醒：不能替代医生诊断

## 🔧 技术架构

### 数据流程
```
用户输入
  ↓
AI识别数据类型和数值
  ↓
验证必选项是否完整
  ↓ (完整)         ↓ (不完整)
输出JSON         提示补充
  ↓
前端parseAndSaveHealthData解析
  ↓
保存到对应数据库集合
  ↓
显示成功提示
```

### 数据库映射
| dataType | 集合名称 | 说明 |
|----------|---------|------|
| bloodTest | bloodTests | 血常规 |
| liverFunction | liverFunctionTests | 肝功能 |
| kidneyFunction | kidneyFunctionTests | 肾功能 |
| bloodSugar | bloodSugars | 血糖 |
| bloodOxygen | bloodOxygens | 血氧 |
| bloodPressure | bloodPressures | 血压 |
| ebv | ebvRecords | EB病毒 |
| cmv | cmvRecords | 巨细胞病毒 |
| ldh | ldhRecords | LDH |

### JSON数据格式规范
```json
{
  "dataType": "数据类型标识",
  "date": "YYYY-MM-DD",
  "values": {
    "指标名小写": 数值,
    "wbc": 3.6,
    "plt": 120
  },
  "notes": "备注信息（可选）"
}
```

**重要说明**：
- `dataType` 必须使用规定的类型标识
- `values` 中的键名必须小写（如 `wbc`、`plt`、`hgb`）
- 日期默认为当天，也可以指定具体日期
- 所有数据自动关联当前档案的 `openid` 和 `profileId`

## ⚠️ 注意事项

1. **必须先登录**：未登录用户无法保存数据
2. **必须有档案**：确保已创建并选择了健康档案
3. **数据格式**：数值类型的指标请输入数字，避免输入文字
4. **日期格式**：系统会自动使用当天日期，无需用户指定
5. **云函数版本**：确保云函数已更新到最新版本

## 🐛 故障排查

### 数据保存失败
1. 检查是否已登录
2. 检查是否选择了档案
3. 查看云函数日志是否有错误
4. 确认数据库集合是否存在

### AI不识别数据
1. 检查是否提供了必选项
2. 尝试使用更标准的表达方式
3. 查看云函数日志中的AI响应内容

### JSON解析失败
1. 查看Console日志中的错误信息
2. 检查AI输出的JSON格式是否正确
3. 确认云函数返回的content字段

## 📊 数据统计

保存的数据会自动包含以下元数据：
- `openid`：用户标识
- `profileId`：档案ID
- `date`：记录日期
- `source: 'ai_assistant'`：数据来源标记
- `createTime`：创建时间戳

这些数据可以在健康档案页面查看，也会在健康图谱中可视化展示。
